1) Product Vision & PRD

ShopVridhi is an offline-first, multi-tenant SaaS platform built for all Indian retail formats. It unifies billing, inventory, GST, udhaar, CRM, WhatsApp engagement, and hyperlocal e-commerce in one modular system.

Key Principles

  • Offline-first with automatic sync
  • Ultra-fast billing and daily operations
  • Custom fields and modules per shop type
  • Marketplace-ready inventory from day one

Business Outcomes

  • Faster billing and fewer stock errors
  • Better udhaar collections and supplier control
  • Higher retention via CRM and loyalty
  • New order channel through pincode discovery

3) User Roles and Permissions

Shop Owner

Full business, finance, settings, and subscription controls.

Manager

Day-to-day operations, reporting, inventory, team oversight.

Cashier

Billing, returns, customer handling, payment collection.

Staff

Stock inward/outward, location updates, basic catalog tasks.

ShopVridhi Admin

Tenant ops, support, plans, compliance and health monitoring.

Customer / Delivery Partner

Order placement/tracking and last-mile execution.

4) Core Modules and Workflows

POS Billing

Barcode/search add -> discount -> split payments -> print/share invoice in seconds.

Item + Barcode

Template-based item creation with tax class, custom fields, and barcode generation.

Inventory Location

Track quantity by rack/shelf/bin with transfers, audits, and variance posting.

Purchase + Supplier

PO -> GRN -> supplier invoice -> payable ledger with due tracking.

CRM + Loyalty

Segment customers, run WhatsApp campaigns, reward repeat purchases.

Udhaar

Auto ledger from invoices, reminders, and settlement tracking.

5) Hyperlocal E-commerce Design

  • Inventory-driven catalog publishing from shop panel
  • Pincode-based customer discovery and shop ranking
  • Order routing with fallback on timeout/rejection
  • Delivery partner API for task assignment and tracking

6) Offline-First Architecture & Sync

PouchDB (Local)CouchDB (Cloud)Conflict ResolverEvent Queue

  • All writes commit locally first; sync replicates in background.
  • Bi-directional replication filtered by tenant/shop.
  • Idempotent event processing to avoid duplicate financial entries.
  • UI always shows sync state: synced / pending / failed.
