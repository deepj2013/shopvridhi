# BA Deep Research Pack: Prioritized Win-vs-Competitor Backlog (Next 3 Sprints)

## Prioritization Method
Score each item by:
- Impact on merchant daily workflow
- Differentiation vs competitor
- Time to deliver
- Revenue/conversion contribution

## Sprint 1 (Speed and Core Reliability)

### Theme
Win cashier and owner trust with speed, accuracy, and daily control.

### P1 Stories
1. Keyboard-first POS with hold/resume and split payment
2. Barcode scan/search with instant item match
3. Day-end cash summary with payment mix and mismatch flags
4. Rack/shelf/bin mapping in item quick-add and stock movement
5. Udhaar ledger auto-post from credit sale and simple recovery reminders
6. Offline sync state indicator in top bar with retry visibility

### Success Metrics
- repeat invoice completion <= 25 sec
- POS error correction rate < 5%
- daily closure usage > 60% active shops

## Sprint 2 (Control and Growth Activation)

### Theme
Beat competitors on inventory intelligence and merchant growth basics.

### P1 Stories
1. Purchase entry with supplier invoice + GST + margin visibility
2. Low-stock, dead-stock, and fast-moving report cards
3. CRM tags (regular/wholesale/VIP/udhaar) + customer purchase history
4. WhatsApp campaign templates with audience filters
5. Public shop page (catalog lite) with shareable link
6. Role controls for owner/manager/cashier/staff with activity logs

### Success Metrics
- purchase posting time reduced by 30%
- low-stock alert action rate > 40%
- campaign send adoption > 20% pilot shops

## Sprint 3 (Commerce and Monetization Edge)

### Theme
Create a clear moat through commerce pipeline and SaaS-grade control.

### P1 Stories
1. Online order intake into POS as draft bill
2. Order accept/reject timer and reroute logic
3. Delivery assignment and status tracking basics
4. Commission rule engine v1 for marketplace orders
5. Universal admin control panel v1 (tenant health, risk controls, plan actions)
6. Compare page + migration assistant flow on public website

### Success Metrics
- order-to-bill conversion > 50% for enabled shops
- delivery status update compliance > 90%
- trial-to-demo conversion uplift > 20%

## Deferred Candidates (Post Sprint 3)
- loyalty points redemption automation
- AI-assisted reorder suggestions
- advanced cohort retention analytics
- settlement dispute automation

## Dependency Notes
- RBAC must precede sensitive admin controls
- data events must be defined before growth experiments
- website compare page requires finalized competitor evidence
