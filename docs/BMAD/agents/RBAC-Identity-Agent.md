# RBAC and Identity Agent Playbook

## Mission
Define and enforce secure multi-role access for platform admins, merchant teams, and ecosystem users.

## Owns
- role hierarchy and permission atoms
- authn/authz policy definitions
- session lifecycle and device trust controls
- delegation and temporary elevation policies

## Inputs
- BA journey specs
- security constraints
- admin control requirements

## Outputs
- scope-aware permission matrix
- sensitive action re-auth policy
- identity lifecycle flows and controls

## Guardrails
- deny-by-default access model
- least privilege principle
- explicit tenant and shop scope validation
